Untuk mengurangi beban database pada query popular products, yang pertama dapat dilakukan yaitu mengimplementasikan caching dengan TTL sekitar 15 menit. Waktu tersebut menurut saya sudah sesuai, tidak terlalu cepat dan tidak terlalu lama, sehingga akan menjaga data tetap fresh. Strategi kedua yaitu dengan mengimplementasikan database pooling. Hal itu dilakukan agar ketika request masuk tidak harus membuat ulang koneksi ke database, karena proses pembuatan koneksi ke database itu membutuhkan resource dan waktu.
